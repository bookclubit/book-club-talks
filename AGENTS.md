# AGENTS.md — инструкции для AI-инструментов

Универсальные правила для любого AI-ассистента (Claude Code, Codex, Cursor,
Antigravity и др.), работающего с этим репозиторием. Логика генерации доклада
намеренно вынесена в детерминированный скрипт — задача ассистента собрать
параметры и **вызвать скрипт**, а не редактировать HTML вручную. Это гарантирует
одинаковый результат независимо от инструмента.

## Что это за репозиторий

Презентации докладов книжного клуба. Каждая папка `BC-*` — самодостаточный
статический доклад, который публикуется на Cloudflare Pages. Все доклады
разворачиваются из **единого шаблона** `_template/` и данных в `data/`.

## Как добавить доклад (главный сценарий)

1. Прочитай данные, чтобы показать пользователю доступные варианты:
   - `data/books.json` — книги и их метаданные (`id`, `code`, `title`, авторы…).
   - `data/books/<id>/chapters.json` — главы и темы книги
     (`chapter_number`, `chapter_title`, `topics: string[]`).
   - `data/speakers.json` — спикеры (`id`, `name`, `surname`).
2. Собери у пользователя (если не задано): книгу, главу, **тему доклада**,
   спикера, номер стрима. Тема — одна из `topics` внутри выбранной главы.
3. Запусти генератор (НЕ редактируй HTML руками):
   ```bash
   node scripts/new-talk.mjs \
     --book <book.id> --chapter <номер> --topic <индекс-с-1|точное-название> \
     --speaker <speaker.id> --stream <номер стрима> [--seq <порядковый>]
   ```
   Скрипт создаст папку `BC-<стрим>-<CODE>-<глава>-<ФАМИЛИЯ>[-<seq>]`, скопирует
   шаблон, подставит книгу/авторов/спикера, соберёт слайды «Программа вечера» и
   «Что далее», скопирует ассеты локально и проставит OG-метатеги.
4. Опубликуй через pull request (см. ниже). Команды печатает сам скрипт.

Человек может запустить то же интерактивно: `npm run new-talk`.

## Публикация — только через pull request

Ветка называется так же, как папка доклада; после одобрения PR доклад
публикуется автоматически (проект Cloudflare Pages создаётся в CI сам).

```bash
git checkout -b <ИМЯ-ПАПКИ>
git add <ИМЯ-ПАПКИ>
git commit -m "<ИМЯ-ПАПКИ> — <тема>"
git push -u origin <ИМЯ-ПАПКИ>
gh pr create --fill --base main
```

На PR автоматически поднимается **превью-деплой** и запускается **lint
автономности** (`npm run lint:talks`). После мержа в `main` — продакшн-деплой.

## Данные (единственный источник правды)

- Новый спикер → добавь запись в `data/speakers.json` и аватар в
  `data/speakers/assets/` (поле `avatar` = `assets/<файл>`).
- Новая книга → запись в `data/books.json` (мета, авторы), главы/темы в
  `data/books/<id>/chapters.json`, ассеты в `data/books/<id>/assets/`
  (`cover/`, `authors/`). `code` — токен для имени папки (`DOCKER`, `REACT`).
- Содержимое `chapters.json` (главы и темы) заполняется по оглавлению книги —
  это контент, которым владеет клуб.

## Правила

- Не редактируй сгенерированный `index.html` доклада ради «хром»-слайдов
  (титул/о книге/программа/что далее) — меняй данные и перегенерируй. Контентные
  слайды (после «Программы») спикер пишет сам.
- Не подключай внешние ресурсы (CDN, шрифты, картинки по URL) — только
  относительные пути внутри папки доклада. Это проверяет `lint:talks`.
- Не переименовывай уже опубликованные папки `BC-*` — сломается ссылка.
- Имена проектов Cloudflare Pages — всегда lowercase (имя папки в нижнем регистре).
