# add-talk

Создаёт новый доклад из единого шаблона `_template/` и данных из **book-club-data**
(единый источник контента; путь — `--data`, env `BOOK_CLUB_DATA` или `../book-club-data`).
Вся генерация — детерминированный скрипт `scripts/new-talk.mjs`; руками HTML не собирай.

## Шаги

1. Прочитай из book-club-data: `index.json` (книги: `folder`/`id`/`chapters`,
   спикеры), `books/<folder>/meta.json` (мета книги, `code`),
   `books/<folder>/chapters/<slug>/chapter.json` (глава `order`/`title`, темы
   `topics[].title`) — покажи пользователю варианты.
2. Уточни (если не задано): книга, глава, **тема доклада** (одна из тем главы),
   спикер, номер стрима, и нужен ли `--seq` (если у спикера в стриме несколько докладов).
3. Запусти генератор:
   ```bash
   node scripts/new-talk.mjs \
     --book <folder|id> --chapter <slug|номер> \
     --topic <индекс-с-1 | id | точное название темы> \
     --speaker <speaker.id> --stream <номер стрима> [--seq <n>] [--data <путь>]
   ```
4. Покажи пользователю напечатанные скриптом имя папки, проект Cloudflare Pages,
   будущий URL и команды публикации через PR (см. навык `open-pr`).

## Что делает скрипт

- Имя папки: `BC-<стрим>-<CODE>-<глава>-<ФАМИЛИЯ>[-<seq>]`.
- Копирует единый шаблон, подставляет книгу/подзаголовок/описание/обложку,
  авторов, выбранного спикера.
- Собирает слайд «Программа вечера» (темы главы: пройденные — зачёркнуты с меткой
  «Пройдено», текущая — активная, следующие — обычные) и «Что далее» (текущая
  становится пройденной, активной — следующая тема). Спикеры на этих слайдах не
  показываются.
- Копирует все ассеты локально в папку доклада, проставляет OG-метатеги.

## Новый спикер или книга

Контент добавляется через **CMS** (`book-club-cms` → pull request в
`book-club-data`), а не в этом репозитории:
- Спикер → форма спикера CMS (имя, аватар, соцсети). `surname` для имени папки
  выводится из `id` (`pomazkov-anton` → `POMAZKOV`).
- Книга → форма книги CMS; для генератора нужен `code` (`DOCKER`, `REACT`).
- Событию-эфиру нужен `stream` (номер стрима) — тоже в форме встречи CMS.

## Важно

- Контентные слайды (после «Программы») пишет спикер — шаблон даёт стиль и
  примеры лейаутов, а не готовый доклад.
- Не подключай внешние ресурсы — только относительные пути (проверяет
  `npm run lint:talks`).
