# add-talk

Создаёт новый доклад из единого шаблона `_template/` и данных `data/`. Вся
генерация — детерминированный скрипт `scripts/new-talk.mjs`; руками HTML
доклада не собирай.

## Шаги

1. Прочитай `data/books.json` и `data/speakers.json`, покажи пользователю
   доступные книги, главы, темы и спикеров.
2. Уточни (если не задано): книга, глава, **тема доклада** (одна из `topics`
   внутри главы), спикер, номер стрима, и нужен ли порядковый номер `--seq`
   (только если у спикера в этом стриме несколько докладов).
3. Запусти генератор:
   ```bash
   node scripts/new-talk.mjs \
     --book <book.id> --chapter <номер главы> \
     --topic <индекс-с-1 | точное название темы> \
     --speaker <speaker.id> --stream <номер стрима> [--seq <n>]
   ```
4. Покажи пользователю напечатанные скриптом имя папки, проект Cloudflare Pages,
   будущий URL и команды публикации через PR (см. навык `deploy-talk`).

## Что делает скрипт

- Имя папки: `BC-<стрим>-<CODE>-<глава>-<ФАМИЛИЯ>[-<seq>]`.
- Копирует единый шаблон, подставляет книгу/подзаголовок/описание/обложку,
  авторов, выбранного спикера.
- Собирает слайд «Программа вечера» (темы главы: пройденные — зачёркнуты с меткой
  «Пройдено», текущая — активная, следующие — обычные) и «Что далее» (текущая
  становится пройденной, активной — следующая тема). Спикеры на этих слайдах не
  показываются.
- Копирует все ассеты локально в папку доклада, проставляет OG-метатеги.

## Новый спикер или книга

- Спикер → добавь в `data/speakers.json` (`id`, `name`, `surname`, `avatar`,
  `url`) и положи аватар в `data/speakers/`.
- Книга → добавь в `data/books.json` (мета, `authors`, `chapters`/`topics`) и
  ассеты в `data/books/<id>/` (`cover/`, `authors/`).

## Важно

- Контентные слайды (после «Программы») пишет спикер — шаблон даёт стиль и
  примеры лейаутов, а не готовый доклад.
- Не подключай внешние ресурсы — только относительные пути (проверяет
  `npm run lint:talks`).
