# 📚 Book Club Talks

Архив презентаций **книжного клуба фронтендеров**. Здесь живут слайды всех
докладов: каждое выступление — отдельная страница, которую можно открыть в
браузере и переслать ссылкой. Публикуются они автоматически, так что после
своего доклада ты за пару минут выкладываешь слайды и делишься ссылкой с клубом.

Это часть экосистемы клуба вместе с `book-club-data`, `book-club-miniapp` и
`book-club-bot`.

## 🎤 Я выступил — как опубликовать свою презентацию

Твоя презентация — это папка с `index.html` (и, если нужно, папкой `assets/`
с картинками, стилями и скриптами). Чтобы она появилась в интернете:

### Шаг 1. Назови папку по формату клуба

```
BC-<номер стрима>-<КНИГА>-<глава>-<ТВОЁ ИМЯ>[-<номер доклада>]
```

- Всё капсом, слова через дефис.
- Последний `-1`, `-2` добавляй, только если у тебя в одном стриме **несколько**
  докладов.

Примеры:

| Папка                     | Что это                                  |
| ------------------------- | ---------------------------------------- |
| `BC-112-DOCKER-8-ANTON`   | Стрим 112, книга Docker, глава 8, Антон  |
| `BC-113-DOCKER-9-ANTON-1` | Стрим 113, глава 9, Антон, доклад 1      |
| `BC-113-DOCKER-9-ANTON-2` | Стрим 113, глава 9, Антон, доклад 2      |

Внутри папки обязательно должен быть **`index.html`** — это стартовая страница
презентации.

### Шаг 2. Положи папку в репозиторий

```bash
git clone git@github.com:bookclubit/book-club-talks.git
cd book-club-talks

# скопируй сюда свою готовую папку с презентацией, например BC-114-DOCKER-10-ANTON
```

### Шаг 3. Создай проект для публикации (один раз на доклад)

Это разовое действие для каждой **новой** папки — говорит хостингу, что появился
новый доклад. Имя проекта = имя папки в нижнем регистре.

```bash
wrangler pages project create bc-114-docker-10-anton --production-branch main
```

> Нужен установленный и авторизованный `wrangler` (`npm i -g wrangler`,
> затем `wrangler login`). Если правишь уже существующую папку — этот шаг пропускай.

### Шаг 4. Запушь — и всё опубликуется само

```bash
git add BC-114-DOCKER-10-ANTON
git commit -m "BC-114 Docker гл.10 — доклад Антона"
git push
```

После пуша автоматика соберёт и опубликует твою презентацию. Через минуту-две
она будет доступна по адресу:

```
https://<имя-папки-в-нижнем-регистре>.pages.dev
```

Например: `BC-114-DOCKER-10-ANTON` → **https://bc-114-docker-10-anton.pages.dev**

Готово — кидай ссылку в чат клуба. 🎉

## 💡 Полезно знать

- **Автономность.** Каждая презентация самодостаточна: пути к картинкам и стилям —
  относительные (`assets/...`), никаких зависимостей от других докладов.
- **Правки.** Обновил слайды в уже опубликованной папке → просто снова `git push`.
  Ссылка та же, содержимое обновится.
- **Не переименовывай** уже опубликованные папки — сломается ссылка, а старый
  проект «осиротеет». Нужна другая презентация — заводи новую папку.
- Список уже опубликованных докладов — в
  [Actions](https://github.com/bookclubit/book-club-talks/actions) и в папках
  `BC-*` этого репозитория.

## 🤖 Работаешь через Claude Code?

В репозитории есть готовые навыки — просто попроси:

- **`add-talk`** — создаст папку доклада с правильным именем и стартовым `index.html`.
- **`deploy-talk`** — заведёт проект и подскажет команды публикации.

Подробное описание устройства репозитория и правил — в [CLAUDE.md](./CLAUDE.md).

## ⚙️ Как это работает под капотом

Публикация делается через [Cloudflare Pages](https://pages.cloudflare.com/).
Workflow [.github/workflows/deploy.yml](./.github/workflows/deploy.yml) при пуше
в `main`:

1. через `git diff` находит, какие папки `BC-*` изменились;
2. деплоит каждую в одноимённый (lowercase) проект Cloudflare Pages.

Аутентификация — секрет репозитория `CLOUDFLARE_API_TOKEN`
(и опционально `CLOUDFLARE_ACCOUNT_ID`). Это уже настроено — обычному спикеру
трогать не нужно.
