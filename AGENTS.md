# AGENTS.md — инструкции для AI-инструментов

Универсальные правила для любого AI-ассистента (Claude Code, Cursor,
Antigravity и др.), работающего с этим репозиторием. Логика генерации доклада
намеренно вынесена в детерминированный скрипт — задача ассистента собрать
параметры и **вызвать скрипт**, а не редактировать HTML вручную. Это гарантирует
одинаковый результат независимо от инструмента.

## Что это за репозиторий

Презентации докладов книжного клуба. Каждая папка `BC-*` — самодостаточный
статический доклад, который публикуется на Cloudflare Pages. Все доклады
разворачиваются из **единого шаблона** `_template/` и **единого источника
контента — репозитория `book-club-data`** (книги, главы, темы, спикеры).

## Как добавить доклад (главный сценарий)

1. Данные берутся из `book-club-data` (путь: `--data <путь>`, env
   `BOOK_CLUB_DATA` или сосед `../book-club-data`). Чтобы показать варианты:
   - `index.json` — книги (`folder`, `id`, `title`, `chapters: string[]`) и
     спикеры (`id`, `name`, `avatar`, `socials`).
   - `books/<folder>/meta.json` — метаданные книги (`title`, `authors`, `cover`,
     `description`, **`code`** — токен для имени папки: `DOCKER`, `REACT`).
   - `books/<folder>/chapters/<slug>/chapter.json` — глава (`order`, `title`,
     `topics: [{id, title, file}]`).
2. Собери у пользователя (если не задано): книгу, главу, **тему доклада**,
   спикера, номер стрима. Тема — одна из `topics` выбранной главы.
3. Запусти генератор (НЕ редактируй HTML руками):
   ```bash
   node scripts/new-talk.mjs \
     --book <folder|id> --chapter <slug|номер> --topic <индекс-с-1|id|название> \
     --speaker <speaker.id> --stream <номер стрима> [--seq <порядковый>] [--data <путь>]
   ```
   Скрипт создаст папку `talks/BC-<стрим>-<CODE>-<глава>-<ФАМИЛИЯ>[-<seq>]`, скопирует
   шаблон, подставит книгу/авторов/спикера, соберёт слайды «Программа вечера» и
   «Что далее», скопирует ассеты локально и проставит OG-метатеги.
4. Опубликуй через pull request (см. ниже). Команды печатает сам скрипт.

Человек может запустить то же интерактивно: `npm run new-talk`.

## Публикация — только через pull request

Ветка называется так же, как папка доклада; после одобрения PR доклад
публикуется автоматически (проект Cloudflare Pages создаётся в CI сам).

```bash
git checkout -b <ИМЯ-ПАПКИ>
git add talks/<ИМЯ-ПАПКИ>
git commit -m "feat(talk): <ИМЯ-ПАПКИ> — <тема>"
git push -u origin <ИМЯ-ПАПКИ>
gh pr create --base main \
  --title "feat(talk): <ИМЯ-ПАПКИ> — <тема>" \
  --body-file .github/pull_request_template.md
```

PR оформляется **в едином формате** по шаблону `.github/pull_request_template.md`
(блок «Доклад» + чеклист). Заполни поля и чеклист; превью-ссылку руками не
добавляй — `preview.yml` сам запостит комментарий со ссылкой. На PR также
запускается lint автономности (`npm run lint:talks`). После мержа в `main` —
продакшн-деплой. Единый навык для Claude Code — `open-pr`.

## Коммиты

- Пиши сообщения коммитов **на русском языке**.
- Формат — [Conventional Commits](https://www.conventionalcommits.org):
  `<тип>(<область>): <краткое описание в повелительном наклонении>`.
- Типы: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`,
  `ci`, `chore`.
- Новый доклад — тип `feat`, область `talk`:
  `feat(talk): BC-112-DOCKER-9-POMAZKOV — Docker в рабочих средах`.
- Другие примеры:
  - `fix(generator): чинит резолвинг путей ассетов`
  - `docs(readme): описывает жизненный цикл доклада`
  - `ci(preview): добавляет комментарий с превью-ссылкой`

## Данные (единственный источник правды — book-club-data)

- Контент (книги, главы, темы, спикеры, картинки) живёт в **`book-club-data`**,
  а не в этом репозитории. Генератор читает его напрямую; отдельного `data/`
  в talks больше нет.
- Картинки — в `book-club-data/media/` (`covers/`, `authors/`, `speakers/`),
  пути в JSON абсолютны от корня (`/media/covers/x.webp`).
- Новая книга/спикер/глава/тема добавляются через **CMS** (`book-club-cms`) —
  она открывает pull request в `book-club-data`.
- Для генерации доклада книге нужен `code` (в `meta.json`), а событию-эфиру —
  `stream` (номер стрима); оба задаются в CMS. Имя папки —
  `BC-<stream>-<code>-<order главы>-<ФАМИЛИЯ спикера из id>`.

## Правила

- Не редактируй сгенерированный `index.html` доклада ради «хром»-слайдов
  (титул/о книге/программа/что далее) — меняй данные и перегенерируй. Контентные
  слайды (после «Программы») спикер пишет сам.
- Не подключай внешние ресурсы (CDN, шрифты, картинки по URL) — только
  относительные пути внутри папки доклада. Это проверяет `lint:talks`.
- Не переименовывай уже опубликованные папки `BC-*` — сломается ссылка.
- Имена проектов Cloudflare Pages — всегда lowercase (имя папки в нижнем регистре).
