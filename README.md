# Book Club Talks

Архив презентаций книжного клуба фронтендеров. Слайды каждого доклада — это
отдельная папка `BC-*` со статической страницей, которая публикуется в вебе
автоматически. После выступления вы добавляете свою презентацию через
pull request; после его одобрения слайды выкладываются и становятся доступны
по ссылке.

## Публикация презентации после доклада

Все доклады оформлены в едином стиле и создаются из общего шаблона одной
командой — вручную верстать титул, слайд о книге и программу не нужно. Публикация
идёт через pull request и не требует ни `wrangler`, ни доступа к Cloudflare.

### Шаг 1. Сгенерируйте каркас доклада

```bash
git clone git@github.com:bookclubit/book-club-talks.git
cd book-club-talks
npm run new-talk
```

Скрипт спросит книгу, главу, тему доклада, спикера и номер стрима (списки берутся
из `data/`) и создаст правильно названную папку
`BC-<стрим>-<КНИГА>-<глава>-<ФАМИЛИЯ>` с готовыми слайдами: титул, «о книге» с
авторами, «Программа вечера» и «Что далее». Пример имени:
`BC-114-DOCKER-10-POMAZKOV`.

> Нет вашей книги/темы/спикера в списках? Добавьте их в `data/` (см.
> [AGENTS.md](./AGENTS.md)) или попросите мейнтейнера. Можно также попросить
> AI-ассистента (Claude Code, Cursor, Codex) — он вызовет тот же генератор.

### Шаг 2. Наполните доклад и запушьте ветку

Отредактируйте контентные слайды (после «Программы») — стиль и примеры лейаутов
уже готовы в шаблоне. Ветка называется так же, как папка доклада.

Коммиты — на русском языке по [Conventional Commits](https://www.conventionalcommits.org)
(`feat(talk): <папка> — <тема>`).

```bash
git checkout -b BC-114-DOCKER-10-POMAZKOV
git add BC-114-DOCKER-10-POMAZKOV
git commit -m "feat(talk): BC-114-DOCKER-10-POMAZKOV — Подготовка к развертыванию"
git push -u origin BC-114-DOCKER-10-POMAZKOV
```

### Шаг 3. Откройте pull request

PR создаётся в едином формате по шаблону (блок «Доклад» + чеклист). В вебе шаблон
подставится сам; командой — так:

```bash
gh pr create --base main \
  --title "feat(talk): BC-114-DOCKER-10-POMAZKOV — Подготовка к развертыванию" \
  --body-file .github/pull_request_template.md
```

Дождитесь зелёных проверок `CI` и `Preview` — бот пришлёт ссылку на превью
комментарием в PR.

### Шаг 4. После одобрения PR публикация происходит автоматически

Как только мейнтейнер одобрит PR и вольёт его в `main`, автоматика соберёт
и опубликует презентацию. Проект хостинга создаётся сам при первой публикации —
никаких ручных действий не требуется.

Через одну-две минуты презентация будет доступна по адресу:

```
https://<имя-папки-в-нижнем-регистре>.pages.dev
```

Например, `BC-114-DOCKER-10-POMAZKOV` → `https://bc-114-docker-10-pomazkov.pages.dev`.

## Полезно знать

- Превью на PR. На каждый pull request автоматически поднимается превью-версия
  доклада, ссылка появляется комментарием в PR — можно посмотреть дек до мержа.
- Автономность. Каждая презентация самодостаточна: пути к ассетам —
  относительные (`assets/...`), никаких внешних CDN. Это проверяет CI
  (`npm run lint:talks`) на каждом PR.
- Правки. Чтобы обновить уже опубликованные слайды, откройте новый PR с
  изменениями в той же папке. После вливания в `main` ссылка остаётся прежней,
  содержимое обновляется.
- Не переименовывайте уже опубликованные папки — это ломает ссылку, а прежний
  проект хостинга остаётся без источника. Для другой презентации заводите новую папку.
- Список опубликованных докладов — это папки `BC-*` в репозитории; статус
  публикаций виден во вкладке
  [Actions](https://github.com/bookclubit/book-club-talks/actions).
