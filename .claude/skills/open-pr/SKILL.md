# open-pr

Открывает pull request для доклада в едином оформлении, одинаковом у всех.

## Предусловия

- Доклад лежит в папке `BC-*` (создан через `add-talk` / `npm run new-talk`).
- `npm run lint:talks` проходит.

## Шаги

1. Ветка должна называться так же, как папка доклада:
   ```bash
   git checkout -b <ИМЯ-ПАПКИ>
   git add <ИМЯ-ПАПКИ>
   git commit -m "feat(talk): <ИМЯ-ПАПКИ> — <тема>"
   git push -u origin <ИМЯ-ПАПКИ>
   ```
   (коммиты — на русском по Conventional Commits, см. `AGENTS.md`).
2. Создай PR по шаблону `.github/pull_request_template.md`:
   ```bash
   gh pr create --base main \
     --title "feat(talk): <ИМЯ-ПАПКИ> — <тема>" \
     --body-file .github/pull_request_template.md
   ```
   Затем заполни в теле поля (папка, книга, глава/тема, спикер) и отметь чеклист.
   Человек может запустить `gh pr create --base main` без `--body-file` —
   редактор откроется уже с этим шаблоном.
3. Не добавляй превью-ссылку руками: workflow `preview.yml` сам поднимет превью и
   запостит/обновит комментарий со ссылкой. Дождись зелёных `CI` и `Preview`.

## Единый формат

- **Заголовок:** `feat(talk): <ИМЯ-ПАПКИ> — <тема>`.
- **Тело:** шаблон `.github/pull_request_template.md` (блок «Доклад» + чеклист + превью).
- **Превью:** только автоматически, комментарием от `preview.yml`.
