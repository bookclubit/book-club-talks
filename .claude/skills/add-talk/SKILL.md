# add-talk

Создаёт новую папку доклада с правильным именем и стартовым `index.html`.

## Соглашение об именовании

Формат папки: `BC-<номер стрима>-<КНИГА>-<глава>-<СПИКЕР>[-<порядковый номер>]`

- Всё капсом, разделитель — дефис.
- Порядковый номер (`-1`, `-2`, …) добавляй, только если у одного спикера
  в одном стриме несколько докладов.
- Примеры: `BC-112-DOCKER-8-POMAZKOV`, `BC-113-DOCKER-9-POMAZKOV-1`.

## Шаги

1. Уточни у пользователя (если не задано): номер стрима, книгу, главу, имя спикера,
   и есть ли у него в этом стриме другие доклады (нужен ли порядковый номер).
2. Собери имя папки по формату выше. Проверь, что такой папки ещё нет.
3. Создай папку и в ней:
   - `index.html` — стартовая страница с названием доклада;
   - `assets/css/`, `assets/js/`, `assets/img/` — по необходимости.
4. Напомни, что публикация идёт через pull request: ветка с именем папки → PR
   в `main` → после одобрения доклад публикуется автоматически (проект
   Cloudflare Pages создаётся в CI сам). См. навык `deploy-talk`.

## Шаблон index.html

```html
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>НАЗВАНИЕ ДОКЛАДА</title>
  <link rel="stylesheet" href="assets/css/style.css" />
</head>
<body>
  <main>
    <h1>НАЗВАНИЕ ДОКЛАДА</h1>
    <p>Спикер · Книга · Глава</p>
  </main>
  <script src="assets/js/main.js"></script>
</body>
</html>
```

## После создания

- Имя проекта Cloudflare Pages для новой папки — её имя в нижнем регистре.
  Например `BC-114-DOCKER-10-POMAZKOV` → `bc-114-docker-10-pomazkov`.
- Публикация произойдёт автоматически после вливания PR в `main`. Отдельно
  создавать проект Cloudflare Pages не нужно — CI сделает это сам при первом деплое.
